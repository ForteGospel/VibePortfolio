# HTML & CSS exercise resources

Reference HTML page plus your own CSS, served in development with **[Vite](https://vitejs.dev/)**.

## Prerequisites

- [Node.js](https://nodejs.org/) (LTS is fine). The **npm** command is included when you install Node — you do not install npm separately in most cases.
- If the terminal says **`npm: command not found`** (or similar), install Node.js from the link above, then **close and reopen the terminal** (or restart the computer) so your system picks up the new tools.
- You can check that things work with:

  ```bash
  node -v
  npm -v
  ```

  Both commands should print a version number.

## Setup

```bash
cd path/to/ex_2_resources
npm install
```

## Development

```bash
npm run dev
```

The terminal prints URLs (Vite’s default port is **5173**). The dev server is set to **open `/elements.html` in your browser** when it starts.


CSS changes usually hot-reload automatically.

## Production build (optional)

```bash
npm run build
npm run preview
```

## Opening without Vite

If you open `elements.html` as a `file://` URL, root paths like `/styles/main.css` will **not** work. Use `npm run dev`, or switch those links to relative paths (`styles/main.css`, `public/favicon.png`) for file-based viewing only.

## What to edit

| Path | Purpose |
|------|--------|
| `elements.html` | Reference markup (headings, lists, forms, Hebrew samples, etc.). |
| `styles/main.css` | **Your styles** — layout, typography, colors. |
| `styles/reset.css` | Shared reset; Do not edit this file. |
| `main.js` | Example script; not linked from `elements.html` by default. |
| `index.html` | Sends the browser to `elements.html` so the site root works in dev. |
| `vite.config.js` | Vite options (dev server, build inputs). |

## Publishing to GitHub Pages

These steps will put your project live at `https://<your-username>.github.io/<your-repo-name>/`.

### 1. Create a GitHub repository

1. Go to [github.com](https://github.com) and sign in.
2. Click **+** → **New repository**, give it a name (e.g. `my-project`), leave it **Public**, and click **Create repository**.

### 2. Update `vite.config.js`

Open `vite.config.js` and set `base` to match your repository name:

```js
base: "/<your-repo-name>/",
```

For example, if your repo is named `my-project`:

```js
base: "/my-project/",
```

### 3. Push your code

In the terminal, inside the project folder:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/<your-username>/<your-repo-name>.git
git push -u origin main
```

### 4. Enable GitHub Pages with GitHub Actions

1. On your repository page, go to **Settings → Pages**.
2. Under **Source**, select **GitHub Actions**.
3. That's it — no need to choose a branch or folder.

### 5. Deploy

Every time you push to `main`, GitHub will automatically build the project and publish it. You can watch the progress under the **Actions** tab of your repository.

Once the workflow finishes (usually under a minute), your site is live at:

```
https://<your-username>.github.io/<your-repo-name>/
```

## What `.gitignore` is for

This repo’s `.gitignore` skips things you should not commit, including:

- **`node_modules/`** — reinstall with `npm install`.
- **`dist/`** — build output from `npm run build`.
- **Logs** and common editor/OS files (e.g. `.DS_Store`).

**Do commit:** your HTML/CSS/JS, `package.json`, `package-lock.json`, `vite.config.js`, and this `README.md`. Do **not** commit secrets.
